# Contributing

1. Create a branch from the default branch: `git switch -c feature/short-name`.
2. Keep secrets and real customer data out of commits and test fixtures.
3. Run `npm run check` and manually test Demo Mode before opening a merge/pull request.
4. Use a clear commit message and explain user-visible changes in the request.
5. Require review for changes to API handling, security headers, or disclaimers.

By contributing, you agree that your contribution is licensed under this
repository's MIT License. Confirm that this license is acceptable under your
employer's policy before publishing.
